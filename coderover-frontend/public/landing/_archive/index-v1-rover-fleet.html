§ Features

Built for the things that actually slow you down.

CodeRover doesn't replace your editor or your CI. It runs alongside them as an autonomous fleet — reading your graph, patrolling your repo, proposing fixes, and remembering context across months. You approve. It ships.

Autonomous PR review

Every pull request gets a first-pass review in under two minutes — style, logic, blast-radius, and test coverage. You read comments, not diffs.

[scout] PR #412 · reviewed in 1.8s · 2 findings

Pre-commit security

Sentinel patrols 24/7 for hardcoded secrets, missing auth, and policy drift. Violations are caught at commit, not after deploy.

[sentinel] BLOCK api_key hardcoded @ src/auth.ts:42

Refactor on autopilot

Tinker finds 200-line functions, circular deps, and dead code — then drafts the refactor PR. You approve or toss. Zero cold starts.

[tinker] proposed 3 refactors · +4% test cov · awaiting review

§ The Fleet

Five specialists. One MCP endpoint.

Not one bloated agent. Five focused ones. Each rover has a narrow job, a tight prompt, and a clean CLI. Mix-and-match or deploy the fleet.

Unit 01

Scout

pr-review agent

Reviews every PR the moment it opens. Flags style, logic, and cross-module blast radius. Runs as a GitHub App or a git hook.

› rover scout --watch
Unit 02

Tinker

refactor agent

Hunts long functions, circular deps, and duplicated logic. Writes the fix as a draft PR. Human approval required.

› rover tinker --propose
Unit 03

Sentinel

security agent

Real-time patrol for secrets, missing auth, PII in logs. Ships with 40+ rules; register your own in YAML.

› rover sentinel --strict
Unit 04

Beacon

health-report agent

Weekly code-health digest: risk modules, trending metrics, merged fixes. Drops into Slack or email.

› rover beacon --weekly
Unit 05

Archive

decision-memory agent

Persistent memory of every architectural decision. New devs ask "why did we drop Redis?" and get the real answer.

› rover archive --ask

Everything dev-friendly.

No vendor lock-in. No "contact sales." Local binary, MCP protocol, plain text logs.

  • Self-hosted by default — runs in your cluster, your laptop, or your CI. Code never leaves.
  • MCP-native — plug into Cursor, Claude Desktop, VS Code, or anything that speaks MCP.
  • Plain-text everything — logs, reports, config. Grep it, pipe it, diff it.
  • Model-agnostic — Claude, GPT, local Llama. Point at any OpenAI-compatible endpoint.
  • Human-in-the-loop — rovers propose, you approve. No silent writes to main.
