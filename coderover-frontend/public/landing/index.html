§ The graph

Not a token pile. A structured map of your repo.

CodeRover parses every file through tree-sitter, lifts symbols and dependencies into Memgraph, and tags every edge with a confidence score. The result: questions RAG can't answer — and answers you can actually trust.

example · auth module subgraph
IMPORTS CALLS INHERITS INFERRED(0.72) · confidence-tagged edge
Query 01

Cycle detection

Find circular import chains of any length. Deterministic, not a fuzzy similarity match — either the cycle exists or it doesn't.

MATCH (a:File)-[:IMPORTS*2..]->(a) RETURN path LIMIT 10; → 2 cycles found · 4 files
Query 02

Impact analysis

"If I change auth.service.ts, what else breaks?" Forward + reverse transitive closure across IMPORTS and CALLS.

MATCH (s:File {name:"auth.service.ts"}) <-[:IMPORTS|CALLS*]-(dep) RETURN DISTINCT dep.name; → 23 files · 47 symbols
Query 03

Hotspots

Files with the highest in-degree. The most depended-on, most fragile, most worth refactoring first.

MATCH (f:File)<-[:IMPORTS]-() RETURN f.name, count(*) AS deps ORDER BY deps DESC; → top: auth.service.ts (41)
what services depend on AuthService?
MATCH (s:Service)-[:CALLS|IMPORTS*1..3]->(:Service {name:"AuthService"}) RETURN DISTINCT s.name, s.file; → 7 services · PrReviewService, IngestService, WatcherService, …

§ Confidence tags

Every edge carries its evidence. No black-box vibes.

Other tools give you an answer. CodeRover gives you the answer plus a tag — so you know whether to trust it, double-check it, or treat it as a candidate list. The same tagging carries through to PR-review findings and RAG citations.

EXTRACTED

Parsed from source.

Deterministic. The symbol appears in the AST, the import statement is in the file, the class extends the base. 100% confident. Used for DEFINES and static IMPORTS.

INFERRED(0.xx)

Deduced from signals.

Cross-file name resolution, dynamic call sites, type inference. The score reflects how many signals agreed. Used for most CALLS and late-binding references.

AMBIGUOUS

Could be one of several.

Overloaded names, duck-typed calls, reflection. CodeRover surfaces all candidates rather than guessing. You (or your agent) pick the right one with full context.

§ The fleet

Five specialists. One MCP endpoint.

Narrow job, tight prompt, real code. Each rover is a module in coderover-api — not a branding exercise. Mix-and-match or run the whole fleet behind the same approval gate.

Unit 01

Scout

[agent-pr] · graph-aware PR review

Pulls the changed-file subgraph, runs deterministic checks (naming, test coverage, blast radius) plus an LLM pass over the diff. Posts findings as GitHub comments.

[agent-pr] PR #412 · subgraph 14 files · 3 findings · 2 confirmed
Unit 02

Tinker

[agent-refactor] · smell detection

Eight detectors — long functions, god classes, circular deps, dead code, duplicated logic, deep nesting, high fan-out, missing tests. Drafts the fix as a PR; human approval required.

[agent-refactor] 3 smells · draft PR opened · awaiting approval
Unit 03

Sentinel

[agent-enforcer] · rule enforcement

Built-in security rules (hardcoded secrets, SQL injection, missing auth) plus custom YAML rules scoped per repo. Runs on every commit via git hook or CI.

[agent-enforcer] BLOCK api_key literal @ src/auth.ts:42
Unit 04

Archive

[agent-memory] · team decisions

Persistent key-value store with TTL. Archives dismissals, precedents, architecture rationale. Queryable by any agent via MCP or the REST API.

[agent-memory] q="why drop redis?" → DR-0032 · signed @maya · 2025-03-14
Unit 05

Beacon

[agent-health] · weekly digest

Scheduled scan across all connected repos — risk modules, trending metrics, merged refactors. Downlinked to Slack or email on your cadence.

[agent-health] downlink · 5 repos · 4 risk modules · armed

§ Under the hood

From git clone to queryable graph in under a minute.

Nine steps. No magic, no black box. Every step is plain-text observable via the admin UI or OTel traces — so when something goes wrong, you can actually see where.

GitHub fetch

Latest commit SHA, changed files (incremental) or all files (force-reindex). OAuth or PAT.

Framework detect

Reads root config — package.json, requirements.txt, build.gradle, next.config.js.

File filter

Excludes node_modules, .git, binaries, lockfiles. Keeps source + artifacts.

AST chunk

Tree-sitter splits on function / class boundaries. Line numbers preserved for citations.

Embed

pgvector with content-hash cache. Unchanged files skip re-embedding. Batched per-repo.

Graph sync

Memgraph ingests symbols + edges with deterministic IDs. Confidence tag per edge.

Artifact index

OpenAPI specs, Terraform, DB schemas get their own artifact table with its own embeddings.

Metadata upsert

Language, framework, module, NestJS role written to code_chunks for filtered search.

Sync log

Commit SHA, file count, chunk count, duration. Next incremental pass starts from here.

§ On your own infra

One docker compose up. Your graph, your keys, your agents.

No SaaS middleman between your source and your LLM. No per-seat pricing. No token margin. Everything the fleet needs — Postgres + pgvector, Memgraph, Redis, the API — ships in one compose file.

Bring your own LLM key

OpenRouter, Anthropic direct, OpenAI, local Llama / Ollama, or any OpenAI-compatible endpoint. Rotate from the admin UI; no redeploy.
